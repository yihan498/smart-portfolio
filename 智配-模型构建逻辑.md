## 一、设计哲学与问题定义

### 1.1 要解决的核心问题

绝大多数个人投资者面临三个层次的失败：

| 失败类型 | 表现 | 根本原因 |
|---|---|---|
| 决策主观化 | 跟风买入、情绪割肉 | 没有系统框架，依赖直觉 |
| 配置同质化 | 抄别人的仓位 | 忽视个体差异，别人的配置≠自己的配置 |
| 比例无依据 | 拍脑袋定20%BTC | 没有目标函数，没有约束条件 |

**智配的目标**：提供一套**个性化的、有数学依据的**资产配置决策工具，将"你是谁"（主观情况）映射为"你应该如何配置"（数学最优解）。

### 1.2 核心设计原则

**原则一：分离主观判断与数学优化**
人的生活状态（年龄、收入、家庭）无法被算法直接读取，需要通过因子打分转化为可量化的"风险预算"。一旦风险预算确定，资产配置由数学决定，不再依赖主观判断。

**原则二：风险承受能力 ≠ 风险承受意愿**
一个人可能"想要"高风险，但"不应该"承担高风险（年龄太大，或资产已经够多）。模型区分这两个维度，通过强约束处理"不应该"，通过打分捕捉"能够"。

**原则三：非线性约束优先于线性叠加**
年龄和财富规模对投资策略的影响是**非线性且具有阶段性的**——它们不仅是影响RS的一个因子，更是覆盖RS的全局约束。线性叠加无法实现这种优先级关系，必须通过独立的上限机制实现。

## 二、整体架构：三层结构

```
个人情况输入
    │
    ▼
┌─────────────────────────────────────┐
│  第一层：11因子RS评分系统             │
│  RS_raw = Σ(wi × Fi)  i=1..11       │
│  捕捉：财务状况 + 生活结构 + 主观意愿  │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  第二层：三层约束覆盖                 │
│  ① 青年加速奖励（年龄<35岁叠加）      │
│  ② 年龄上限（时间维度硬约束）         │
│  ③ 财富保全上限（规模维度硬约束）      │
│  RS_final = min(RS_raw+①, ②, ③)    │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  第三层：均值-方差优化（MVO）          │
│  RS_final → σ_target（目标波动率）   │
│  在 σ_p ≤ σ_target 约束下           │
│  max w·μ → 最优配置比例              │
└─────────────────────────────────────┘
    │
    ▼
最优资产配置：BTC / 美股 / 中国资产 / 防守
```

**三层结构的分工**：

- **第一层（RS打分）**：捕捉你现在能承担多少风险（财务能力维度）
- **第二层（约束覆盖）**：确定你现在应该承担多少风险（生命周期维度）
- **第三层（MVO优化）**：在给定风险预算下，数学求解最优配置（效率优化维度）

## 三、RS指标的构建

### 3.1 RS指标的定义与意义

**RS（Risk Score，综合风险承受得分）**是一个0-10的连续指标，综合衡量一个人的**客观风险承受能力**。

RS的设计目标是：
- **单一数字**：将11个维度的信息压缩为一个数，便于后续与MVO对接
- **可解释性**：每个维度的贡献可追溯
- **个性化**：相同年龄、不同财务状况的人得到不同RS
- **动态性**：市场环境（F8、F9）随时间变化，RS随之更新

### 3.2 11个评估因子的选择逻辑

因子的选择覆盖三个维度：**财务能力**、**生活结构**、**外部环境**。

#### 财务能力因子（合计55%权重）

| 因子 | 权重 | 计算公式 | 设计逻辑 |
|---|---|---|---|
| **F1 年龄** | 20% | `max(0, (65-age)/40 × 10)` | 越年轻时间跨度越长，亏损后有更多时间恢复；25岁满分，65岁归零 |
| **F2 收入储蓄比** | 15% | `min(10, (收入-支出)/收入 × 20)` | 储蓄率越高，在不动用本金的前提下，承受亏损的能力越强；储蓄率50%=满分 |
| **F3 资产充裕度** | 10% | `min(10, 总资产/年支出/2)` | 资产可覆盖的生活年数越多，无需被迫变现的缓冲越大；20年支出=满分 |
| **F4 紧急储备金** | 10% | `min(10, 应急金/(月支出×1.2))` | 没有应急金则可能被迫低点抛售；12个月生活费=满分 |
| **F5 负债压力** | 10% | `max(0, 10 - 负债/资产 × 10)` | 高负债意味着强制还款压力，降低投资灵活性 |

> **F1的权重为什么最高（20%）？**
> 年龄是财务能力中最基础的约束：它直接决定了投资时间跨度，而时间跨度是能否在亏损后恢复的根本条件。一个70岁的人即使有大量资产，也缺乏等待长周期反弹的时间窗口。

#### 生活结构因子（合计20%权重）

| 因子 | 权重 | 选项 | 设计逻辑 |
|---|---|---|---|
| **F6 家庭负担** | 10% | 10/8/6/4/2（单身→多孩） | 抚养义务越重，投资决策牵一发动全身，承受亏损的容忍度越低 |
| **F7 收入稳定性** | 5% | 10/8/6/4/2（国企→创业） | 收入不稳定时，波动性投资可能叠加现金流危机 |

#### 外部环境因子（合计10%权重）

| 因子 | 权重 | 选项 | 设计逻辑 |
|---|---|---|---|
| **F8 市场周期** | 5% | 10/8/6/4/2（熊底→牛顶） | 熊市底部是提高风险敞口的最佳时机；牛市顶部应压缩风险 |
| **F9 经济周期** | 5% | 8/6/4/2（扩张→衰退） | 经济扩张期资产质量整体较高，衰退期应增加防御 |

#### 主观判断因子（合计10%权重）

| 因子 | 权重 | 选项 | 设计逻辑 |
|---|---|---|---|
| **F10 投资经验** | 5% | 10/7/5/3/1（>10年→<1年） | 经验不足者难以在深度回撤时坚守，可能在低点情绪化抛售 |
| **F11 主观风险意愿** | 5% | 1-10滑块 | 尊重个体主观偏好，但权重较低以防止过度高估自己的承受能力 |

### 3.3 权重设计原则

权重分配遵循以下原则：

1. **客观财务能力优先**：F1-F5合计55%，主观意愿（F11）仅5%。这确保配置主要由客观财务情况驱动，而非情绪。

2. **F11的权重刻意压低**：研究表明，人们在牛市中系统性高估自己的风险承受能力，在熊市中系统性低估。将F11权重限制在5%，防止主观意愿主导配置。

3. **市场环境因子的作用**：F8+F9合计10%，确保同一个人在市场不同阶段会得到不同RS，体现"逆周期配置"的逻辑。

### 3.4 RS原始得分公式

```
RS_raw = 0.20×F1 + 0.15×F2 + 0.10×F3 + 0.10×F4 + 0.10×F5
       + 0.10×F6 + 0.05×F7 + 0.05×F8 + 0.05×F9 + 0.05×F10 + 0.05×F11
```

其中每个Fi ∈ [0, 10]，RS_raw ∈ [0, 10]。

## 四、三层约束覆盖机制

### 4.1 为什么需要约束覆盖，而不是只依赖RS打分？

**问题**：F1（年龄）在RS中只占20%权重。考虑一个80岁的富裕退休者：
- F1（年龄）= 0（满减）→ 贡献 0
- F3（资产充裕度）= 10（资产丰富）→ 贡献 1.0
- F4（紧急储备）= 10 → 贡献 1.0
- F5（负债）= 10（无负债）→ 贡献 1.0
- F6-F11 合理估计 ≈ 3.5

**结果**：RS_raw ≈ 5.5 → 模型推荐平衡型配置，含大量美股。

这显然是错误的：80岁老人无论多富裕，都需要以**资本保全**为核心目标，激进配置的风险无法被时间分散。

**根本原因**：线性加权无法表达"一票否决"类的约束。年龄和财富规模在特定区间内对配置策略具有**主导性**，这种主导性不能被其他因子的高分"稀释"。

解决方案是在RS原始计算之外，**独立设计上限约束**，通过取最小值实现强制覆盖。

### 4.2 第一层：青年加速奖励

**逻辑**：年轻（<35岁）是一种独特的资产——时间跨度极长，即使遭受50%的亏损，也有充足的时间在后续增长中弥补。这种"时间资产"在财务指标中难以直接量化，但对投资决策至关重要。

同时，个人财富积累通常从较低基数开始，年轻时承担更高风险是加速财富积累的有效路径（假设风险管理得当）。

**公式**：
```
youth_boost = max(0, (35 - age) / 35 × 1.5)   当 age < 35
            = 0                                  当 age ≥ 35
```

| 年龄 | 青年加速奖励 |
|---|---|
| 20岁 | +0.64 |
| 25岁 | +0.43 |
| 30岁 | +0.21 |
| 35岁+ | 0 |

**注意**：青年加速奖励会被年龄上限和财富上限抑制——如果一个25岁的人已经拥有大量财富，财富保全上限会压制这个加速奖励，确保逻辑自洽。

### 4.3 第二层：年龄上限（时间维度约束）

**逻辑**：随年龄增加，以下三个因素同时变化：

1. **投资时间跨度缩短**：无法等待长周期反弹（BTC一轮完整牛熊周期约4年）
2. **收入替代能力下降**：退休后无法用劳动收入补充亏损
3. **生活刚需支出优先**：大额亏损直接影响生活质量而非只是账面数字

**公式**：
```
age_cap = max(0.5, 10 × ((max(0, 80 - age) / 55))^0.7)
```

使用0.7次幂（而非线性）确保：
- 40岁以前约束较宽松（cap≥8，保留充分进攻空间）
- 55-65岁约束明显（cap在4-6区间，强制过渡到平衡）
- 75岁以上约束极强（cap<2，接近全保守）

| 年龄 | RS年龄上限 | 对应最激进配置 |
|---|---|---|
| 40岁 | 8.0 | 进攻型 |
| 50岁 | 6.5 | 积极型 |
| 55岁 | 5.8 | 积极型 |
| 65岁 | 4.0 | 平衡型 |
| 70岁 | 3.0 | 稳健型 |
| 75岁 | 1.9 | 保守型 |
| 80岁 | 0.5 | 近全防守 |

### 4.4 第三层：财富保全上限（规模维度约束）

**核心问题**：年龄约束解决了"时间"维度，但没有解决"规模"维度。

一个25岁拥有2000万资产（年支出10万，即200年资产）的人：
- 年龄上限 = 10（不限制）
- 青年加速 = +0.43
- 若RS_raw = 7.5，RS_adjusted = 7.93

但这是合理的吗？他已经实现了财务自由，未来无论发生什么都不会影响生活质量。这时的目标不再是"积累"，而是"保全"——他不需要再冒险了。

**边际财富效用递减原理**：

财富从0增加到100万，对生活质量提升巨大；从1000万增加到2000万，提升边际效用极低。反过来，从2000万损失1000万，痛苦远超从100万损失50万。这意味着，随财富增加，**风险的绝对危害上升，而承担风险的必要性下降**。

**"资产年数"指标**：

衡量财富丰裕程度不用绝对金额，而用**总资产 ÷ 年支出**，即资产可支撑多少年生活开支。这一指标自动考虑了生活成本差异：

- 年支出5万的人，200万资产 = 40年 → 成长期
- 年支出30万的人，200万资产 = 6.7年 → 积累期

**财富保全上限曲线（四阶段）**：

| 财富阶段 | 资产年数 | RS财富上限 | 目标导向 |
|---|---|---|---|
| 积累期 | ≤ 20年 | 10（无约束） | 全力积累，接受高波动 |
| 成长期 | 20–50年 | 10 → 5（线性） | 增长与保全并重 |
| 守成期 | 50–100年 | 5 → 3（线性） | 保全为主，适度增长抗通胀 |
| 传承期 | ≥ 100年 | 3 → 2（线性，最低2） | 资本保全，考虑代际传承 |

**公式**：
```
asset_years = 总资产 / 年支出

wealth_cap =
  如果 asset_years ≤ 20:  10
  如果 20 < asset_years ≤ 50:  10 - 5 × (asset_years - 20) / 30
  如果 50 < asset_years ≤ 100: 5 - 2 × (asset_years - 50) / 50
  如果 asset_years > 100: max(2, 3 - (asset_years - 100) / 100)
```

**为什么最低保留2而非0？**
即使是最富裕的投资者，也需要对抗通货膨胀（长期约3-4%/年）。RS=2对应的配置仍包含约15%美股+10%中国资产，能提供适度的实际回报，防止财富被通胀侵蚀。

### 4.5 三层约束的综合逻辑

```
RS_final = min(
  RS_raw + youth_boost,   ← 11因子得分 + 青年加速（最高10）
  age_cap,                ← 年龄上限
  wealth_cap              ← 财富上限
)
```

**三层约束的互动**：

| 人群 | 起主导作用的约束 | 逻辑 |
|---|---|---|
| 25岁，资产10万 | 无约束（RS原始决定） | 年轻低资产，全力积累 |
| 25岁，资产2000万 | **财富约束**（压至≤2） | 已财务自由，保全优先 |
| 55岁，资产200万 | **年龄约束**（压至≤5.8） | 接近退休，压缩风险 |
| 55岁，资产3000万 | **财富约束**（压至≤3） | 守成期，保全>年龄约束 |
| 70岁，资产500万 | **年龄约束**（压至≤3） | 老年，时间维度主导 |
| 70岁，资产5000万 | **财富约束**（压至≤2.5） | 财富更丰裕，保全更强 |

## 五、均值-方差优化（MVO）

### 5.1 现代投资组合理论基础

均值-方差优化由Markowitz于1952年提出，核心思想是：**在给定风险水平下，存在一个使期望收益最大的最优资产组合（有效前沿）**。

**四类资产的收益-风险特征**：

| 资产 | 预期收益μ | 波动率σ | Sharpe比率 | 作用定位 |
|---|---|---|---|---|
| 比特币 BTC | 20% | 75% | 0.213 | 分散化工具（低相关性） |
| 美股优质个股 | 10% | 18% | **0.333** ★ | 核心增长引擎 |
| 中国高分红/国债 | 7% | 13% | 0.231 | 中期分散化 |
| 美元存款/货基 | 4% | ≈0% | 0.000 | 流动性储备 |

无风险利率 RF = 4%（美元计价）

**相关系数矩阵**：

```
              BTC    美股   中国   防守
BTC          1.00   0.30   0.15   0.00
美股         0.30   1.00   0.35   0.00
中国资产     0.15   0.35   1.00   0.00
防守         0.00   0.00   0.00   1.00
```

BTC与其他资产的低相关性（ρ=0.15-0.30）是其进入组合的关键原因——即使其Sharpe比率低于美股，低相关性带来的组合层面的分散化效益仍使其有价值。

### 5.2 MVO的优化问题表达

```
目标函数：max   w · μ         （最大化期望收益）

约束条件：
  √(w'Σw) ≤ σ_target         （组合波动率不超过目标值）
  Σwᵢ = 1                    （权重之和为1）
  wᵢ ≥ 0                     （不允许做空）
  w_BTC ≤ 40%                 （BTC最大仓位限制）
  w_US ≤ 70%                  （美股最大仓位限制）
  w_CN ≤ 55%                  （中国资产最大仓位限制）
```

其中Σ为资产协方差矩阵 `Σ_ij = σ_i × σ_j × ρ_ij`。

### 5.3 RS → 目标波动率的映射

RS是一个相对数（0-10），MVO需要的是绝对的目标波动率σ。映射公式为：

```
σ_target = 1% + 34% × (RS / 10)^1.5
```

使用1.5次幂（凸函数）确保：
- 低RS区间：σ增长缓慢（保守区间对波动率高度敏感）
- 高RS区间：σ增长加快（进攻区间有更大的波动空间）

| RS | σ_target | 参照系 |
|---|---|---|
| 0 | 1% | 近现金 |
| 2 | 4.0% | 纯债券组合 |
| 5 | 13.0% | 经典60/40股债组合 |
| 7 | 20.9% | 纯股票组合（~美股） |
| 10 | 35% | 激进配置 |

**为什么用幂函数而非线性？**
人对风险的感知和效用函数是非线性的（即行为金融学中的前景理论）。低风险区间每增加1%波动率，对保守投资者的心理冲击远大于对激进投资者。幂函数映射使RS间隔在主观体验上更均匀。

### 5.4 MVO查找表的预计算

由于MVO是一个二次规划问题，在页面实时计算需要较大计算量，因此采用**预计算101点查找表**的方式：

- RS从0.0到10.0，步长0.1，共101个档位
- 每档位用Python的`scipy.optimize.minimize`（SLSQP方法）求解
- 采用7个不同初始点，取最优解，确保全局最优
- 结果存储为JSON格式内嵌于HTML，查询时间O(1)

## 六、MVO与RS的结合合理性

### 6.1 为什么这种结合是合理的？

传统的资产配置模型要么纯主观（顾问建议），要么纯数学（忽视个人情况）。智配的创新在于**将个性化主观评估与数学优化在同一框架下整合**：

**两步分工的关键优势**：

| 步骤 | 解决的问题 | 为什么这步由人/数学完成 |
|---|---|---|
| RS评分（人） | 你能承受多少风险？ | 生活情况无法被算法自动读取 |
| MVO优化（数学） | 这些风险应该如何分配？ | 有严格的数学最优解，主观判断反而劣化结果 |

**对比其他方法**：

- **纯问卷方法**：只有RS评分，没有MVO，配置比例主观确定，缺乏数学依据
- **纯MVO方法**：参数固定，忽视个人风险承受差异，一刀切
- **智配方法**：RS决定风险预算，MVO决定预算内的最优分配

### 6.2 美股为什么始终是最大仓位？

这是数学优化的必然结果，而非人为设定。原因：

**美股Sharpe比率最高（0.333）**：每承担一单位风险，美股提供最多的超额收益。当MVO在约束条件下最大化收益时，会优先配置Sharpe比最高的资产，直到达到上限（70%）或波动率约束绑定。

**BTC的价值不在于最高收益，而在于分散化**：
- BTC期望收益20%，看似最高，但σ=75%，Sharpe仅0.213（低于美股）
- BTC的价值在于ρ_BTC,US=0.30（低相关）：加入BTC使组合在相同σ下能提升预期收益
- 在高RS区间，BTC逐渐增加至40%，是分散化效益驱动的，而非单纯追逐高收益

**防守资产的归零时机**：
- 当σ_target足够高，防守资产（Sharpe=0）占用波动率预算但不贡献超额收益
- RS≥6时，优化结果中防守资产趋于0，是数学最优而非主观判断

### 6.3 三层约束与MVO的互动

三层约束通过**限制RS_final的范围**间接影响MVO的输入σ_target：

- 同一个人，RS_final=3 → σ_target=6.6% → 保守配置（55%防守资产）
- 同一个人，RS_final=7 → σ_target=20.9% → 进攻配置（无防守资产）

约束层不改变MVO的数学结构，只改变"进入MVO之前的风险预算"，保持了两层系统的独立性和可维护性。

## 七、完整计算流程

```
输入：age, income, expenses, assets, emergency, liabilities
      F6(家庭), F7(稳定性), F8(市场), F9(经济), F10(经验), F11(意愿)

─────────────────────────────────
STEP 1：计算各因子得分

F1 = max(0, min(10, (65-age)/40 × 10))
F2 = min(10, max(0, (income-expenses)/income × 20))
F3 = min(10, assets/expenses/2)
F4 = min(10, emergency/(expenses/12)/1.2)
F5 = max(0, 10 - liabilities/assets × 10)
F6~F11 = 直接从输入读取

─────────────────────────────────
STEP 2：计算RS原始分

RS_raw = 0.20×F1 + 0.15×F2 + 0.10×(F3+F4+F5+F6)
       + 0.05×(F7+F8+F9+F10+F11)
RS_raw = clamp(RS_raw, 0, 10)

─────────────────────────────────
STEP 3：三层约束覆盖

youth_boost = age<35 ? max(0,(35-age)/35×1.5) : 0

age_cap = max(0.5, 10×((max(0,80-age)/55)^0.7))

asset_years = assets/expenses
wealth_cap =
  asset_years ≤ 20  → 10
  20 < ay ≤ 50  → 10 - 5×(ay-20)/30
  50 < ay ≤ 100 → 5 - 2×(ay-50)/50
  ay > 100      → max(2, 3-(ay-100)/100)

RS_final = min(min(RS_raw + youth_boost, 10), age_cap, wealth_cap)
RS_final = clamp(RS_final, 0, 10)

─────────────────────────────────
STEP 4：RS → 目标波动率

σ_target = 1% + 34% × (RS_final/10)^1.5

─────────────────────────────────
STEP 5：MVO查表

key = round(RS_final, 1)  → 查询预计算表
返回：{btc%, us%, cn%, def%, 预期收益%, 组合σ%, Sharpe}

─────────────────────────────────
输出：最优配置比例 + 组合统计指标
```

## 八、模型局限性与使用建议

### 8.1 参数估计的不确定性

| 参数 | 不确定性来源 | 影响 |
|---|---|---|
| μ（预期收益） | 前瞻估计，历史均值外推 | MVO对μ的敏感性高，微小变化可导致权重显著变化 |
| σ（波动率） | 基于历史，未来可能更高或更低 | BTC历史数据约10年，估计误差较大 |
| ρ（相关系数） | 在危机期间往往趋向1 | 极端市场中分散化效益大幅减弱 |

### 8.2 MVO的已知缺陷

1. **正态分布假设**：MVO假设收益率服从正态分布，但BTC等资产存在显著肥尾（极端涨跌概率远高于正态）
2. **参数敏感性**：MVO是"误差放大器"——输入参数的小误差会被放大为配置权重的大变化
3. **单期模型**：MVO是单期最优，不考虑再平衡成本和税务影响

### 8.3 RS指标的已知局限

1. **线性加权假设**：假设各因子独立且线性叠加，实际上因子间存在交互效应（如：高负债+低收入的组合比两者单独的影响之和更危险）
2. **F3的双重性质**：资产充裕度在RS中越高越好（能承担风险），但在财富保全层越高越受限（不需要冒险）。两者同时作用形成正确的非线性行为，但可能使F3对最终RS的净影响不直观
3. **主观因子的自报偏差**：F11由用户自报，牛市中系统性高估，熊市中系统性低估

### 8.4 使用建议

| 场景 | 建议 |
|---|---|
| 常规使用 | 每年完整重新评估一次 |
| 市场周期变化 | 单独调整F8，即时更新配置建议 |
| 重大生活事件 | 结婚、生育、退休、大额负债等事件后立即重新评估 |
| 极端市场 | 模型建议作为决策起点，结合个人判断和专业意见 |
| 绝对上限 | 任何单一资产配置不建议超过模型上限（BTC≤40%，美股≤70%） |

**本模型定位**：量化决策辅助工具，提供系统性框架和数学最优解作为参考基准。最终投资决策须由本人做出，本工具不构成投资建议。
